# CSS 深度修改样式
前端开发中使用UI库开发，大大的节省了开发的时间以及人力等等成本。但是有点不好的是，有时候一些UI库的默认样式达不到我们需要的效果，这个时候就需要我们自己手动改写一些相关的样式了。然后，问题来了，怎么去改固定的样式？也许有人会选择自己定义一些样式文件，替换掉原始的样式 —— 这是个不错的方法，也是个相当常用的方法。然而如果需要修改的位置不是很多的话，博主个人感觉这个方法不是很方便。所以这里博主列出几个比较常用的深度修改CSS的方法，仅供参考，如果有其他的也欢迎继续补充~~~

<!-- more -->

* **/deep/**

如果是使用预编译语言（Eg: `Sass`、`Scss`、`Stylus`) 编写样式的话，可以考虑使用 `/deep/` 进行深层次修改操作。
举个栗子：

```markdown
.classOne /deep/ .classTwo
```

如上面的栗子，其中的 `.classOne` 为自己定义的一个类名，在需要进行深层次修改的类名 （`.classTwo`） 外层定义，确保编译之后的 `.classTwo` 位于 `.classOne` 的内部，即可实现对 `.classTwo` 原样式的重定义操作。


* **>>>**
如果是使用原生的 CSS 语言进行样式编写的话，则可以使用 `>>>` 进行深层次的修改操作。
举个栗子：

```markdown
.classOne >>> .classTwo
```

如上面的栗子，其中的 `.classOne` 为自己定义的一个类名，在需要进行深层次修改的类名 （`.classTwo`） 外层定义，确保编译之后的 `.classTwo` 位于 `.classOne` 的内部，即可实现对 `.classTwo` 原样式的重定义操作。
